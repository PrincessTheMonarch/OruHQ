* Simplicity

---

# 📄 PRODUCT PRD (UPDATED SYSTEM)

## Product Name

OruHQ

## Product Type

Freelancer Operating System

## Core Purpose

OruHQ helps freelancers manage:

* Projects
* Tasks (simple to-do lists, not Kanban)
* Invoices
* Payments
* Earnings insights

Everything is connected but flexible — users can start from Projects, Invoices, or Payments and link them together.

---

# 🧠 CORE PRODUCT RULES (VERY IMPORTANT)

1. No Kanban boards anywhere in the system
2. Tasks must be simple checkbox to-do lists (Notion-style)
3. Projects are the central entity
4. Financial clarity is more important than workflow complexity
5. Everything must be readable in under 5 seconds
6. Lists > Cards > Charts (in priority order)

---

# 🧩 CORE MODULES

## 1. Projects (Core System)

Users can:

* Create projects
* Add project name
* Add optional client name
* Set deadline
* Set budget

Each project contains:

* Simple task list (checkbox system only)
* Progress indicator (based on completed tasks)
* Linked invoices (optional)
* Linked payments (optional)

---

## 2. Tasks (UPDATED SYSTEM)

Replace any Kanban or board UI with:

### Task Structure:

* Task title
* Checkbox (done / not done)
* Optional due date
* Optional notes

### Behavior:

* Tasks are grouped under projects
* No drag and drop
* No status columns
* Completion = binary system
* Progress is automatically calculated

---

## 3. Invoices

Users can:

* Create invoices
* View all invoices
* Track status:

  * Draft
  * Sent
  * Paid
  * Overdue

Invoices can be:

* Standalone
* Linked to projects (optional)

Users can also:

* Create a project from an invoice

---

## 4. Payments

Users can:

* Create payments manually
* Track payment status:

  * Pending
  * Completed

Payments can be:

* Standalone
* Linked to invoices
* Linked to projects
* Linked to tasks

---

## 5. Settings (Invoice Configuration)

Users must be able to configure invoice defaults:

* Business name
* Business email
* Address
* Logo
* Currency
* Invoice prefix (e.g. ORU-001)
* Payment terms (7/14/30 days)
* Default footer note

All invoices should auto-fill from settings.

## 4. AT-RISK PROJECTS (LIST ONLY)

Replace any alert boxes or banners.

Use a simple list format:

Each item includes:

* Project name

* Subtle status indicator

No red-heavy UI. Keep it minimal.

## 5. TASK OVERVIEW (NOTION-STYLE LIST)

* Title
